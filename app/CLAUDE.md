# app ‚Äî React Native Android (MindSync)

RN 0.76+ with **New Architecture enabled** (Fabric + TurboModules).
Android only. TypeScript strict. Zustand + op-sqlite + Drizzle ORM.

> **This workspace is scaffolded in milestone M1** (see `../BUILDPLAN.md`).
> Until then, `app/package.json` and `app/android/` don't exist yet.

## Run / build (post-M1)
- `pnpm android` ‚Äî debug build on connected device (requires adb + device in dev mode)
- `pnpm android:release` ‚Äî signed release APK ‚Üí `android/app/build/outputs/apk/release/`
- `pnpm start` ‚Äî Metro bundler (auto-started by `pnpm android`)
- `adb logcat -s ReactNative:V ReactNativeJS:V MindSync:V` ‚Äî filtered logs

## Directory contract
- `src/native/` ‚Äî TS wrappers around Kotlin TurboModules. One file per
  module, matching the Kotlin class name (e.g., `NotificationListener.ts`
  wraps `NotificationListenerModule.kt`).
- `src/db/` ‚Äî Drizzle schema in `schema.ts` over op-sqlite. Migrations
  generated by `drizzle-kit generate` into `src/db/migrations/`. Bundled
  into the APK and applied on app boot via `migrate()`.
- `src/stores/` ‚Äî One Zustand store per domain (`useSyncStore`,
  `usePermissionsStore`, etc.). Stores don't import each other.
- `src/services/sync/` ‚Äî background sync. Reads from DB, POSTs to server,
  advances cursor in DB. Idempotent ‚Äî server dedupes by `(source, externalId)`.

## Android native (Kotlin)
- Package: `com.mindsync.app`
- Modules live in `android/app/src/main/java/com/mindsync/modules/`
- After adding a new native module: `pnpm android` runs codegen automatically
  (`react-native-codegen`).
- NotificationListenerService requires user-granted access via
  `Settings.ACTION_NOTIFICATION_LISTENER_SETTINGS` intent.
- SMS + call log require runtime `READ_SMS` / `READ_CALL_LOG` permissions.
- Manifest permissions live in `android/app/src/main/AndroidManifest.xml`.

## Patterns
- Never block the JS thread on native calls ‚Äî all TurboModules return Promises.
- Functional components + hooks. No class components.
- Side effects in `useEffect` or Zustand middleware, not in render.
- Import shared types: `import { SmsEnvelope } from '@mind-sync/shared'`.
- **Native Kotlin never writes to SQLite directly.** It emits events to JS.
  JS (via Drizzle) is the single writer ‚Äî keeps migrations simple and
  avoids two-process concurrency bugs.

## Gotchas
- **Metro + pnpm symlinks**: if `@mind-sync/shared` fails to resolve, check
  `metro.config.js` ‚Üí `watchFolders` includes `../packages/shared` and
  `../../node_modules`. Don't add `nohoist`; Metro config is the lever.
- New Architecture: old `NativeModules.X` pattern is deprecated. Use the
  codegen-generated spec + `TurboModuleRegistry.getEnforcing()`.
- **Drizzle + op-sqlite**: use the **sync** driver for hot writes
  (notification capture). The async driver adds microtask overhead that
  matters at 100+ writes/sec during historical SMS/call backfill.
- Run migrations at app boot via Drizzle's `migrate()` ‚Äî before any Zustand
  store touches the DB. Boot order: open DB ‚Üí migrate ‚Üí hydrate stores.
- Don't add iOS files. There is no iOS target.
